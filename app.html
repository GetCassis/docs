Using the Cassis web app

The Cassis web app lives at app.getcassis.com. Sign in with Google, or set a password through "Reset password" on the sign-in screen.

The app has five sections in the left sidebar: Chat, Ontology, Suggestions, Evals, and Tables. Below is a short tour of each, using a Formula 1 project as the running example.

Chat

Ask a question in natural language. For simple questions, Cassis answers directly. For complex or expensive ones, it surfaces a proposed query plan first: a plain-language description of how it will answer, the assumptions it had to make, and the ontology objects it will use. Review the plan, then click Run this plan to execute.

Every answer carries 👍 / 👎 with a free-text comment and an "Add as test case" shortcut. When you correct the agent in chat (a wrong answer, a clarification on a field, an object description that needs fixing), it's captured automatically and surfaced in Suggestions for review.

1. Simple question
Clear questions run directly. Cassis returns the answer in plain language, with the SQL and result so anyone downstream can verify.
Cassis · Formula 1 project
Who won the most races in 2023?
Max Verstappen won 19 out of 22 races in 2023, far ahead of Pérez (2) and Sainz (1).
sql: SELECT d.full_name, COUNT(*) FROM result r
  JOIN race ra ON r.race_id = ra.race_id
  JOIN driver d ON r.driver_id = d.driver_id
  WHERE ra.year = 2023 AND r.position = 1
  GROUP BY d.full_name ORDER BY 2 DESC

results: 3 rows
objects_used: Race Result, Race, Driver

Ontology

Where you explore and edit what Cassis knows about your data. The tab bar at the top has six views: Context, Objects, Definitions, Links, Graph, and Import / Export. Everything is editable inline.

Context
The business context Cassis uses to ground every answer. Season numbering, FR vs global terminology, product conventions, anything an analyst would otherwise explain in a meeting.
Objects
The business objects we extracted from your tables. Each one has a description, its source table, synonyms, and properties.
Definitions
Metric and concept definitions that don't belong to a single object (e.g. "podium", "DNF", "win percentage").
Links
Relationships between objects, with the SQL join pattern and notes.
Graph
Visual overview of the ontology. Useful to spot connections or gaps at a glance.
Import / Export
Export your ontology into Cassis's format for now. More formats come later.

Evals

A dataset of questions with expected answers, plus runs that execute every test case against the current ontology. Add a test case directly from the bottom of any chat with "Add as test case", or create one from scratch in this section.

Test Cases
A question plus its expected answer. The expected answer can be a value or a reference SQL query. Since the underlying data may change, the eval focuses on whether the generated query is right, not whether it returns a specific number.
Runs
A full run across every test case. Review each one's plan, the generated SQL, and the verdict (passed / failed). Use runs to track ontology quality over time.
